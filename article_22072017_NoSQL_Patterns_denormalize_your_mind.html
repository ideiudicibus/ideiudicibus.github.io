The Relational Data approach.

In the RDBMS world that is relational databases, the data modeling processes typically starts by modeling data as a series of tables, consisting of rows and columns, which define the so called schema of your data. Relational database theory has defined  a number of ways of putting application data into tables, referred to as normal forms.  Therefore, understanding modeling related concepts is key to work on any data related project whether is SQL or NoSQL base.Here' I'll cover all of them with simple definitions.


  1. First Normal Form. This makes sure there is no data redundancy and a primary key is well defined on the table. For example, we are not capturing multiple values in a single column (such as comma separated) or using multiple columns instead of rows to capture data with varying fields/identifiers.
  2. Second Normal Form. This makes sure that all columns in the table have 1-1 direct relationship with the entire primary key of the table. Attributes/Columns that likely describe key column of the table can be stored separately. Foreign key is defined to be able to join the multiple tables generated as a result of this type of normalization.
  3. Third Normal Form.This makes sure that any columns describing the non key columns are stored separately and is a higher level of normalization.
